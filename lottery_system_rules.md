# 彩票分析系统规则文档

## 📋 核心规则（必须遵守）

### 1. 试机号/开机号使用规则 ⭐⭐⭐
**重要程度：最高**

- ✅ 试机号和开机号**仅作为参考依据**
- ❌ **严禁将试机号或开机号直接作为开奖号码推荐**
- ❌ 精选2注、10注大底**不得包含**试机号和开机号
- ✅ 必须在页面上醒目显示当天的试机号和开机号（黄色高亮）

**实现代码：**
```python
# 设置排除列表
excluded_numbers = [today_test, today_boot]
# 生成推荐时必须检查
if nums not in excluded_numbers:
    return nums
```

---

### 2. 重点和值规则 ⭐⭐⭐
**重要程度：最高**

- ✅ 每天从**0-27中挑选5个最可能**的和值
- ✅ 精选2注和10注大底的和值**必须从这5个重点和值中产生**
- ❌ 不得超出重点和值范围

**实现代码：**
```python
# 定义重点和值（每天5个）
key_sum_values = [8, 10, 13, 15, 17]  # 排列三示例
key_sum_values = [9, 11, 13, 16, 18]  # 福彩3D示例

# 生成号码时检查和值
if sum(nums) in key_sum_values:
    return nums
```

---

### 3. 精选2注与大底关系 ⭐⭐⭐
**重要程度：最高**

- ✅ **精选2注必须包含在10注大底中**
- ✅ 10注大底中用⭐标记精选2注
- ✅ 精选2注用特殊颜色高亮显示

**实现代码：**
```python
# 先生成精选2注
selected_2 = generate_top2()

# 10注大底必须包含精选2注
base_10 = selected_2 + generate_remaining_8()
```

---

### 4. 战绩统计规则 ⭐⭐
**重要程度：高**

- ✅ 真实记录从**2026年4月1日**开始
- ✅ 不使用模拟数据
- ✅ 每日推荐锁定后等待开奖对比

---

### 5. 10注大底规则 ⭐⭐
**重要程度：高**

- ✅ 大底从30注改为**10注**
- ✅ 和值必须符合重点和值
- ✅ 不得包含试机号和开机号
- ✅ 精选2注必须包含其中

---

### 6. 页面显示规则 ⭐⭐
**重要程度：中**

- ✅ 重要原则显示当天的试机号和开机号（黄色高亮）
- ✅ 排列三和福彩3D的历史记录分开显示
- ✅ 添加禁用缓存meta标签，避免浏览器缓存问题

## 📊 数据流程

```
1. 获取今日试机号、开机号
   ↓
2. 设置排除列表（试机号、开机号）
   ↓
3. 分析历史数据，确定5个重点和值
   ↓
4. 生成精选2注（符合重点和值，排除试机号开机号）
   ↓
5. 生成10注大底（包含精选2注，符合重点和值）
   ↓
6. 推送到GitHub，自动禁用缓存
```

---

## ⚠️ 常见错误及解决方案

| 错误 | 原因 | 解决方案 |
|------|------|---------|
| 推荐号码=试机号/开机号 | 未设置排除 | 添加excluded_numbers检查 |
| 和值不在重点范围 | 未验证和值 | 添加sum(nums) in key_sum_values检查 |
| 精选2注不在大底中 | 独立生成 | 先生成精选2注，再补全大底 |
| 浏览器缓存旧内容 | 未禁用缓存 | 添加meta标签禁用缓存 |
| 历史记录未分开显示 | 数据混在一起 | 分开渲染两个表格 |
